## **PROPÓSITO**
Un proceso de desarrollo de software tiene como propósito la producción eficaz y eficiente de un software que reúna los requisitos del cliente.

## **DEFINICIONES**

*Proceso de software: conjunto de actividades necesarias para transformar los requisitos de un usuario en un sistema software.*
*Necesidad de definir un proceso para posibilitar la repetición, facilitar la realización del mismo por diferentes personas, para llevar a cabo mediciones y mejoras” (Estándar IEEE 1074)*
*Es aquel en que las necesidades del usuario son traducidas en requerimientos de software, estos requerimientos transformados en diseño y el diseño implementado en código, el código es probado, documentado y certificado para su uso operativo". Concretamente "define quién está haciendo qué, cuándo hacerlo y cómo alcanzar un cierto objetivo" [Jacobson 1998].*

**El Proceso software establece el marco técnico y de gestión para asignar métodos, herramientas y personas a las tareas a realizar para construir software.**

## **ETAPAS DEL PROCESO GENÉRICO DEL DESARROLLO DE SOFTWARE**
Aunque existen muchas metodologías, casi todas cumplen estas etapas mínimas:

1) **ESPECIFICACIÓN DE SOFTWARE:** Se define la funcionalidad y las restricciones del sistema (obtención y análisis de requisitos).
2) **DISEÑO DEL SISTEMA:** construye el software de acuerdo a la especificación.
3) **IMPLEMENTACIÓN:** Es la fase de codificación donde se produce el código ejecutable.
4) **VALIDACIÓN:** Se asegura que el sistema cumpla con lo que el cliente quiere mediante pruebas unitarias, de integración y de sistema.
5) **INSTALACIÓN:** Entregar el sistema al usuario y asegurar su operacionalidad.
6) **EVOLUCIÓN Y MANTENIMIENTO:** Cambiar/adaptar el software según las demandas; reparar fallos en el sistema cuando sean descubiertos.
### **CARACTERÍSTICAS DEL PROCESO**
- **ENTENDIBLE:** : Se encuentra el proceso bien definido y es entendible.
- **VISIBLE:** El proceso es visible al exterior.
- **SOPORTABLE:** Puede el proceso ser soportado por herramientas CASE.
- **ACEPTABLE:** El proceso es aceptado por aquellos involucrados en él.
- **CONFIABLE:** Los errores del proceso son descubiertos antes de que se conviertan en errores del producto.
- **ROBUSTO:** Puede continuar el proceso a pesar de problemas inesperados.
- **MANTENIBLE:** Puede el proceso evolucionar para cumplir con los objetivos organizacionales.
- **RAPIDEZ:** Que tan rápido puede producirse el sistema.
### **PROBLEMAS EN EL MODELO DEL PROCESO**
- Normalmente, las especificaciones son incompletas o anómalas.
- No existe una distinción precisa entre la especificación, el diseño y la manufactura.
- Solo hasta que el sistema se ha producido se puede probar.
- El software no se puede remplazar siempre durante el mantenimiento.
### **ACTIVIDADES PROTECTORAS**
- Seguimiento y control de proyecto de software.
- Revisiones técnicas formales.
- Garantía de calidad del software.
- Gestión y configuración del software.
- Gestión de riesgos.
